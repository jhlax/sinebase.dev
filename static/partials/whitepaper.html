Home

Welcome to sinebase. This is a web application that acts as utility program for music production. This website will be as simplistic and un-aesthetic as possible until all of the background logic is configured. In the mean-time, it is extremely experimental and should be treated as such.

Feature Lists

Features available (or planned) are listed here.

Application Feature Wish-List

  • BPM information calculator
  • Logarithmic repitch calculator
  • Project external files analysis
  • Automated project renders organization
  • Project version controls
  • SoundCloud-like streaming (but better)
  • Max for Live integration
  • Special use-case soundfile generators

System Feature Wish-List

  • Markdown notes, journals, and articles
    This is the same as the comment formatting in Reddit, and the README formatting on Github.
  • Real-time analysis of audio
    Using the CFFI Python libraries in addition to the C programming language. Possibly with JACK audio connection kit.
  • Chromatic audio analysis
    This will deliver insight on the theoretical side of music production and composition.
  • Artificially-intelligent sample classification
    The coup-de-gras of sample library organization and the ultimate "what do I need to do next?" option.

Backend Information

The backend of this web application consists of a few languages, and a lot of libraries.

Python

For the bulk of the backend operations (as well as a form of "glue", the Python programming language will be used.) The web application is served by a Flask web server. In order to keep the amount of duplicate requests to a minimum, this application will be a single-page application with a RESTful API communication layer.

C

The C programming language will be used to perform any low-latency or high-speed processing or analysis.
